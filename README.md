  <span>Start Your Live Server</span>
</h2>

Move into your project folder:

```bash
cd ~/storage/emulated/0/your-project-folder
```

> [!important]
> Make sure the project path matches your actual storage location.
> If the path is incorrect, Live Server may fail to serve your files and the preview won't load.

Start server:

```bash
live-server
```

Example output:

```text
Serving ".../your-project-folder"

http://127.x.x.x:8080
```

Copy the generated URL.

  <svg xmlns="http://www.w3.org/2000/svg" width="24" height="24" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round" class="lucide lucide-bookmark-icon lucide-bookmark"><path d="M17 3a2 2 0 0 1 2 2v15a1 1 0 0 1-1.496.868l-4.512-2.578a2 2 0 0 0-1.984 0l-4.512 2.578A1 1 0 0 1 5 20V5a2 2 0 0 1 2-2z"/></svg>
  <span>Session Behavior</span>
 </h2>

**_Responsive Live Server_** remembers your URL while Acode remains open.

Example:

```text
http://127.0.0.1:8080
```

- Exit **_Responsive Live Server_** → URL stays saved
- Reopen **_Responsive Live Server_** → URL automatically restored

However:

- Closing Acode completely
- Force stopping Acode
- Restarting the application

will clear the session URL.

The URL popup will appear again next time.
